Welcome to Harness FME Workshop!

Workshop customers take the express route to toggling feature flags.  

  • Find one of four feature flags in the project and environment you created during installation.
  • Open the corresponding HTML page created during installation.
  • Learn how to toggle the flag.
  • Run an experiment, end-to-end.

If you haven’t configured and installed Workshop, you can find its repository: https://github.com/dbmartin00/fme-workshop You will need account admin rights to create the API key necessary.


What flags are included in Workshop?

magic_boxes.html

This page and flag lets you play with toggling different users, the results displayed visually.  Each “magic” box is tied to the key of a specific end user, (e.g. “f0”).  Harness FME knows how to allocate by key, even randomly.  You can toggle an individual key to turn on a feature with precision, include a segment of keys, and even set percentage allocation to randomly distribute the feature across your boxes.

modal_madness.html

Harness FME features dynamic config, which allows passing a JSON configuration  to the application when a feature flag is tested.  Using the “madness” of the modal page is the ability to change the configuration of the page without changing any code.

form_follies.html

Form follies illustrates multi-variant flags.  You can toggle through the various form layouts to show how easily interface can be tested, like in an ABC test.

enigma.html

As the owner of a travel agency, you’re encouraging your traffic to book rooms.  Madrid is a good destination, but will users like noon, sunset, or night in Madrid?  Each comes with its own image.  

To run an experiment, set your experiment settings loosely, allowing Fixed Horison, 0.2 significance, and smallest possible sample size.  Now you allocate the three treatments equally and reload.  Keep reloading and click the button occasionally.   You should cycle through the three treatments.  After about one hundred simulated users, you can reculate the enigma flag.  It takes a few minutes to obtain results.

The enigma page is testing a feature flag, which generates an impression.  The page has built-in performance metrics that will attach and report automatically.  For a few simulated users, the result is modest, but it counts as a complete experiment.


Final Notes

Workshop is meant to act as a catalyst for your own flags.  enigma, modal_madness and form_follies are good source code references.  Feel free to create your own flags and explore.
